# Changelog

The canonical changelog lives in the repository: **[CHANGELOG.md on GitHub](https://github.com/getbetweenrows/betweenrows/blob/main/CHANGELOG.md)**. It is updated with every release and is the authoritative record of what changed between versions.

Released versions are published as git tags — see **[Tags](https://github.com/getbetweenrows/betweenrows/tags)**. Each tag produces a Docker image at `ghcr.io/getbetweenrows/betweenrows:<version>`.

## Release cadence

During alpha, releases are cut when there is something worth shipping. Versions follow semantic versioning, but pre-1.0 breaking changes can happen in minor version bumps. **Read the changelog before upgrading** — see [Upgrading](/operations/upgrading) for the safe upgrade procedure.
